## Проект 1 — Регрессионное тестирование сервиса Mesto

## Описание проекта
В рамках данного проекта было проведено регрессионное тестирование веб-сервиса **Mesto** на основе подготовленного набора тест-кейсов.  
Цель тестирования — проверить стабильность работы ключевого функционала приложения и убедиться, что ранее реализованные функции работают корректно после изменений.

Во время тестирования выполнялась проверка функциональности интерфейса, фиксировались результаты прохождения тест-кейсов и оформлялись баг-репорты при обнаружении дефектов.

---

## Цели проекта
- Изучить функциональность сервиса **Mesto**
- Провести регрессионное тестирование по готовым тест-кейсам
- Зафиксировать результаты выполнения тестов
- Описать найденные дефекты в виде баг-репортов

---

## Что было сделано
В ходе выполнения проекта были выполнены следующие действия:

1. Изучен список тест-кейсов для регрессионного тестирования.
2. Создана копия таблицы с тест-кейсами для работы.
3. Выполнено тестирование приложения **Mesto** на тестовом стенде.
4. Для каждого тест-кейса был проставлен статус выполнения:
    - `PASSED` — тест выполнен успешно
    - `FAILED` — тест выявил ошибку
    - другие статусы указаны по необходимости
5. Обнаруженные дефекты были оформлены в виде баг-репортов.

---

## Баг-репорты
При обнаружении ошибок были составлены баг-репорты, содержащие:

- краткий и понятный заголовок
- приоритет ошибки
- шаги воспроизведения
- ожидаемый результат
- фактический результат
- скриншоты или другие вложения (во внешних ссылках)

---

## Используемые инструменты
В процессе работы использовались следующие инструменты:

- **Google Sheets** — для ведения тест-кейсов и баг-репортов
- **Браузер** — для тестирования веб-приложения
- **Скриншоты** — для фиксации найденных дефектов
